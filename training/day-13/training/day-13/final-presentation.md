## スライド 5: AI全体像 + リサーチでのAI活用（Day4-5）

### AI基礎（Day4）
- Token = AIが文章を処理する最小単位、Context = 一度に読める限界値
- LLM → RAG → AI Agent へと進化する技術レイヤー
- ハルシネーション：AIが自信満々にウソをつく現象 → 一次ソースでの裏取り＋他AIツールでWチェックが必須

### リサーチ実務（Day5）
- Deep Researchで HR Tech 統合調査レポートを作成
- 発見：時間はかかるが、かなり詳しいレポーティング。参照元リンク付きで確認もできる
- 気づき：専門的な内容はAIの説明をさらにAIで噛み砕いて理解する必要がある。AIが勝手に解釈で情報を取ってくるリスクがあるので、自身でリンクに飛んで確認する作業は必須

## スライド 7: データ分析でのAI活用（Day9）

SaaS KPIデータ（MRR・新規顧客数・解約率・NPS）をAIで分析

### 主要発見
- **2倍**: MRR成長（482万→995万円）
- **r = -0.97**: NPS-解約率の逆相関
- **+5.2%**: 月次平均成長率

### AIの誤り発見
営業担当別累計成約数をAIが誤算 → 田中209/佐藤206/鈴木204が正解

学び：**AIの出力は必ず人間が検証する。数値は特に注意**

## スライド 8: Project A — 競合比較メモまとめツール（Day10-11）

- **課題**: 競合メモがSlack・スプレッドシート等に散在 → 整理に毎回30-60分
- **解決策**: テキスト貼付 → AIが比較表＋強み/弱みを自動生成
- **技術**: Next.js + Claude API + Vercel（Cursorで開発）
- **開発時間**: 設計1時間 + 実装6時間 = 計約7時間
- **ROI**: 月6.7h削減 → 年間80.4時間 → 約24.1万円/年の効果
- **公開URL**: https://project-a-delta-nine.vercel.app/

## スライド 9: Project B — Sirius 展示会LP（Day12）

- **課題**: 展示会来場者にSiriusの価値を30秒で伝えるLPが必要
- **解決策**: 7セクション構成のスマホファーストLP（Claude Codeで開発）
- **開発時間**: 設計1.5h + 実装2h = 計約3.5時間（Aの半分以下）
- **公開URL**: https://project-b-psi-gold.vercel.app/

### Cursor vs Claude Code の使い分け

| 観点 | Cursor | Claude Code |
|------|--------|-------------|
| 向いている場面 | ロジック重視・細部の微調整 | 全体構成を一気に形にする |
| 強み | Diff表示で安心レビュー | 自律的にファイル作成→テスト→修正 |
| 注意点 | 先祖返りが発生することも | ブラックボックス化しやすい |

## スライド 13: まとめ

> **AIは万能ではないが、使い方次第で非エンジニアでも開発・分析・資料作成の質とスピードを劇的に変えられる**
> — 13日間の学びを一言で

### 注意事項（実務移行で守ること）
- **AI出力の確認**: 経歴ミス・古い情報・画像の不自然さは必ず目視チェック
- **機密情報**: 個人名・未発表予算等はAIに入力しない（一般化して入力）
- **品質管理**: AI生成物は「下書き」。自社らしさの調整は人間が仕上げる

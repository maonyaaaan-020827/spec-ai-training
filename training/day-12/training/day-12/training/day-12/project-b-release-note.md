## Cursor AI vs Claude Code の違い

* **良かった点**
    * **「並走感」が最強:** Cursor Tab（AIによる予測補完）が極めて優秀で、次に書きたいコードを先回りして提示してくれるため、タイピングの負荷が劇的に減る。
    * **視覚的な安心感:** 差分（Diff）がエディタ上でハイライト表示されるため、AIの提案を「どの行がどう変わるか」を厳密にレビューしてから反映（Accept）できる。
    * **GUIによる直感操作:** ファイル参照（@）やドキュメントのインデックス化が簡単で、既存コードをベースにしたロジック構築（競合比較ツールのロジック実装など）に向いている。
* **やりにくかった点**
    * **「丸投げ」への不安:** 複雑な命令を出すと、以前修正した箇所を元に戻してしまう「先祖返り」がたまに発生するため、人間が常に監視しておく必要がある。
    * **コンテキストの管理:** プロジェクトが巨大になると、どのファイルをチャットに含めるかを手動で微調整する手間が発生することがある。

## Claude Code での開発体験
* **良かった点**
    * **圧倒的な「完遂力」:** ターミナルから「〜を作って」と指示するだけで、ファイル作成、テスト実行、エラー修正を自律的に繰り返す「エージェント型」の動きが強力。
    * **プロジェクト全体の把握能力:** 特定のファイルだけでなく、ディレクトリ構造全体を俯瞰して修正をかける能力が高く、LPのように複数のファイル（HTML/CSS/JS）を跨ぐ構成を一気に作るのが得意。
    * **自律的なツール実行:** `npm install` やビルドコマンドをAI自らが必要に応じて実行するため、環境構築の手間がほぼゼロになる。
* **やりにくかった点**
    * **ブラックボックス性:** ターミナル上で一気に処理が進むため、途中でどのような試行錯誤が行われたかの詳細を後から追いかけるのが少し難しい。
    * **微調整のインターフェース:** 「このボタンの余白をあと2px右に」といった視覚的な微調整は、エディタ上で直接コードをいじれる Cursor の方が手軽に感じる。

## どちらが自分に合っていたか、その理由

今回の比較では、**「Claude Code」**の方がより「AI時代の開発」という新しい衝撃を感じられました。

**【理由】**
「競合比較まとめツール」のようなロジック重視のものは、開発者が主導権を握って細かく指示を出す **Cursor** が適していると思いました。しかし、今回 **LP作成** を行った際、構成・コーディング・依存関係の解決までを「一言のプロンプト」で完結させられる **Claude Code の自律性**は、開発のスピード感が格段に違いました。

「自分でハンドルを握って細部までこだわりたい時は **Cursor**」、「目的地を伝えて最短距離で形にしたい時は **Claude Code**」という使い分けが、現状のベストアンサーだと感じました。

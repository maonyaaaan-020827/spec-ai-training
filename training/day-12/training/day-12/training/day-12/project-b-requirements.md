# 要件定義書：Sirius 展示会来場者向けサービス紹介LP

**プロジェクト名：** Sirius Exhibition Landing Page
**作成日：** 2026年4月10日
**ステータス：** ドラフト
**技術スタック：** Next.js + Tailwind CSS（データベース不要）
**想定工数：** 1日（8時間以内）

---

## 1. 背景

Siriusは「AI人材評価・配置最適化プラットフォーム」として、企業の人事データを「管理」から「意思決定」へ進化させるサービスである。

現在、多くの中堅企業が以下の課題を抱えている。

- **100名の壁：** 組織拡大により現場の実態が経営層に届かなくなる構造的な情報の非対称性
- **タレマネツールの死蔵：** 入力が更新されず、グラフだけで「答え」がなく、結局は上司の主観に回帰してしまう
- **主観評価の限界：** 人材の本当の価値が可視化されず、適切な配置・評価ができない

展示会に出展するにあたり、来場者にSiriusの価値を短時間で直感的に伝えるためのLPが必要となった。ブースでの名刺交換後にQRコードで誘導する想定であり、スマホでの閲覧が主となる。

---

## 2. ゴール

- **来場者が30秒以内に「Siriusが何をするサービスか」を理解できる** こと
- **「主観の霧をAIが晴らす」** という核心メッセージが一目で伝わるビジュアルを実現すること
- 来場者が興味を持った場合に、すぐに問い合わせ・資料請求のアクションが取れること
- 展示会当日のブース誘導ツール（QRコード経由）として機能すること

## 3. スコープ

### 作るもの

- シングルページのLP（Next.js App Router、静的エクスポート対応）

## 4. ターゲットユーザー

**主要ターゲット：** 展示会来場者（HR Tech系イベントなどを想定）

具体的なペルソナは以下の3層。

- **人事部門の責任者・担当者：** データに基づく意思決定で経営層への提案力を向上させたい。現行のタレマネツールに不満を感じている層
- **現場マネジャー：** 部下の評価に主観が入りがちで、客観的な根拠がほしいと感じている層
- **経営層・役員：** 100名超の組織で人材の実態が見えにくくなっており、経営判断に直結する人材データを求めている層

**利用環境：** 展示会場でスマートフォンから閲覧（QRコード経由）。Wi-Fi環境が不安定な場合も想定し、軽量なページとする。

## 5. 主要機能

### 機能1：ヒーローセクション（ファーストビュー）

Siriusのコアメッセージ「人事データは『管理』から『意思決定』へ」を大きなタイポグラフィとビジュアルで表示。CTAボタン（資料請求 / お問い合わせ）を配置し、初見3秒で興味を引く設計とする。

### 機能2：課題提起 → ソリューション提示フロー

「100名の壁」「タレマネツールの死蔵」「主観評価の限界」をアニメーション付きカードで提示し、その直後にSiriusによる解決策（5ステップの統合フロー：既存DB → データ連携 → AI Engine → 分析・解釈 → 意思決定支援）をビジュアルで表示する。

### 機能3：2-Layer Evaluation（独自基準）の可視化

Siriusの差別化ポイントである「統計的エビデンス × 自社固有の文脈」の2層評価を、インタラクティブな図解で表現。Sirius StandardとCustom Contextが重なり「納得感」が生まれる様子をビジュアルで伝える。

### 機能4：三方よし（ベネフィット）セクション

人事部門・現場マネジャー・社員それぞれへの価値を、アイコン付きカード形式で表示。タップまたはホバーで詳細が展開される仕様とする。

### 機能5：CTA / お問い合わせセクション

ページ末尾に「『なんとなく』の評価を終わらせよう。」というメッセージとともに、資料請求・デモ依頼・お問い合わせの各導線を配置。展示会限定の特典やキャンペーンがあれば併記できるようにする。

## 6. 画面構成

以下の7セクションで構成するシングルページレイアウト。

```
┌─────────────────────────────────────┐
│  ヘッダー（固定）                      │
│  ロゴ + ナビゲーション + CTAボタン       │
├─────────────────────────────────────┤
│  Section 1: ヒーロー                  │
│  キャッチコピー + サブコピー + CTA      │
│  背景：グラデーション or 抽象ビジュアル   │
├─────────────────────────────────────┤
│  Section 2: 課題提起                  │
│  「100名の壁」「ツール死蔵」「主観回帰」  │
│  3カード横並び（モバイル: 縦積み）       │
├─────────────────────────────────────┤
│  Section 3: ソリューション             │
│  5ステップフロー図                     │
│  既存DB → 連携 → AI → 分析 → 意思決定  │
├─────────────────────────────────────┤
│  Section 4: 独自基準（2-Layer）        │
│  ベン図風ビジュアル                    │
│  Sirius Standard × Custom Context    │
├─────────────────────────────────────┤
│  Section 5: ベネフィット（三方よし）     │
│  人事 / マネジャー / 社員 の3カード     │
├─────────────────────────────────────┤
│  Section 6: 競合比較                  │
│  比較テーブル（簡潔に優位性を表示）      │
├─────────────────────────────────────┤
│  Section 7: CTA / お問い合わせ         │
│  「なんとなくの評価を終わらせよう。」     │
│  資料請求 / デモ依頼 / メール           │
├─────────────────────────────────────┤
│  フッター                             │
│  会社情報 + SNSリンク + コピーライト     │
└─────────────────────────────────────┘
```

### セクション詳細

| # | セクション名 | 内容 | 主要コンポーネント |
|---|------------|------|-----------------|
| Header | 固定ヘッダー | ロゴ、セクションリンク、CTAボタン | Navbar（スクロール時に背景色変化） |
| 1 | ヒーロー | コアメッセージ + CTA | 大型タイポグラフィ、CTAボタン×2 |
| 2 | 課題提起 | 3つの課題カード | アニメーション付きカード（Intersection Observer） |
| 3 | ソリューション | 5ステップのフロー | ステップインジケーター、アイコン付き説明 |
| 4 | 独自基準 | 2-Layer Evaluationの図解 | SVGベン図 or CSS図解 |
| 5 | ベネフィット | 三方よしの価値提示 | 3カラムカード（展開式） |
| 6 | 競合比較 | ポジショニング表 | 比較テーブル（チェックマーク形式） |
| 7 | CTA | 最終アクション誘導 | フォームリンク、ボタン群 |
| Footer | フッター | 会社情報 | リンク一覧、コピーライト |

### 技術的な補足

- **フレームワーク：** Next.js 14+（App Router）、`output: 'export'` による静的サイト生成
- **スタイリング：** Tailwind CSS v3+、カスタムカラーパレット（Siriusブランドカラーに準拠）
- **アニメーション：** Framer Motion または CSS Intersection Observer によるスクロール連動アニメーション
- **ホスティング：** Vercel（既存の `sirius-lp.vercel.app` を活用想定）
- **フォント：** Noto Sans JP（Google Fonts）
- **画像：** SVGアイコン・図解を中心に軽量化。写真素材は最小限とし WebP 形式で配信
- **パフォーマンス目標：** Lighthouse Performance 90+ 、ページ全体 500KB 以下
